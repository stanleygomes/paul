## When to apply

Apply this skill whenever you are asked to:

- Create or modify database schemas
- Generate or run migrations
- Integrate with Drizzle ORM
- Move database-related code between apps

Keywords: `database`, `migration`, `drizzle`, `orm`, `postgresql`, `schema`, `table`, `index`

## Conventions

### Migrations

- **MANDATORY**: Use descriptive names for migration files.
- Do **NOT** use the default random names generated by `drizzle-kit` (e.g., `0000_tranquil_terrax.sql`).
- **Format**: `XXXX_descriptive_name.sql`
- **Examples**:
  - `0000_initial_schema.sql`
  - `0001_add_password_hash_to_users.sql`
  - `0002_add_prompt_table.sql`
  - `0003_add_core_tables.sql`

If you use `npx drizzle-kit generate`, you MUST rename the generated file to follow this convention immediately after generation.

### Schema Location

- Database schemas should live in `src/schemas/database/`.
- Export all schemas from a central `index.ts` in that same directory.

### Naming

- Use `snake_case` for table names and column names in the database.
- Use `PascalCase` or `camelCase` for the TypeScript constants representing those tables, following the established practice in the file.

## Checklist

- [ ] Migration files follow the `XXXX_descriptive_name.sql` pattern.
- [ ] Schemas are exported from `src/schemas/database/index.ts`.
- [ ] Foreign keys and indexes are explicitly defined where needed.

## Self-Improvement

- If the user corrects a database convention or migration pattern, **you must update this SKILL.md file** immediately to reflect the new preference.
